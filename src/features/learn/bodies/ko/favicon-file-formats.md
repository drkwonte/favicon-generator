파비콘을 생성하면 파일 하나만 만들어지는 게 아닙니다. 서로 다른 형식과 크기로 이루어진 파일 묶음이 만들어집니다. 각 형식에는 고유한 역할이 있으며, 차이를 이해하면 파비콘이 모든 환경에서 올바르게 표시되도록 만들 수 있습니다.

---

## 파비콘의 세 가지 주요 형식

### ICO — 파비콘의 원조 형식

`.ico` 형식은 파비콘의 시작점입니다. 1999년 인터넷 익스플로러 5와 함께 파비콘 개념이 도입될 때 등장했으며, 20년이 넘는 시간 동안 파비콘의 핵심 형식으로 자리잡아 왔습니다.

ICO만의 특징은 하나의 `.ico` 파일 안에 **여러 크기의 이미지를 동시에 담을 수 있다**는 점입니다. 일반적으로 16×16, 32×32, 48×48 픽셀이 하나의 파일에 포함됩니다. 브라우저는 표시되는 위치에 따라 가장 적합한 크기를 자동으로 선택합니다. 이 다중 해상도 패키징 방식 덕분에 ICO는 오랫동안 기본 파비콘 형식으로 사용되어 왔으며, 특히 구형 브라우저와의 호환성을 보장하는 데 유리합니다.

**ICO를 사용하는 경우:**
- 사이트 루트에 `favicon.ico`를 기본 파비콘으로 배치할 때
- 구형 브라우저 및 오래된 운영체제와의 호환성이 필요할 때
- 다른 형식이 지원되지 않을 때를 대비한 폴백(fallback)으로

**한계:**
- SVG처럼 무한히 확대할 수 없음
- 같은 크기의 PNG에 비해 파일 용량이 큰 편
- 고해상도(레티나) 디스플레이 대응이 단독으로는 부족함

---

### PNG — 현대의 표준 형식

PNG는 오늘날 모던 브라우저에서 가장 널리 사용되는 파비콘 형식입니다. ICO와 달리 PNG 파일 하나에는 특정 크기의 이미지 하나만 담을 수 있기 때문에, 모든 사용 환경을 커버하려면 여러 크기의 PNG 파일이 필요합니다.

완전한 파비콘 셋업에는 일반적으로 다음과 같은 PNG 크기가 포함됩니다.

| 크기 | 용도 |
|------|------|
| 16×16 | 표준 브라우저 탭 |
| 32×32 | 작업 표시줄 바로가기, 고해상도 탭 |
| 180×180 | Apple Touch Icon (iOS 홈 화면) |
| 192×192 | 안드로이드 크롬 홈 화면 |
| 512×512 | 안드로이드 크롬 스플래시 화면, PWA |

PNG는 **완전한 투명도**를 지원합니다. 덕분에 흰색이나 단색 배경의 사각형이 아닌, 형태에 맞는 깔끔한 아이콘 모양을 표현할 수 있습니다. 정사각형이 아닌 형태의 아이콘에 특히 중요합니다.

**PNG를 사용하는 경우:**
- 모던 브라우저의 모든 파비콘 용도에
- Apple Touch Icon (아이폰, 아이패드 홈 화면 바로가기)
- 웹 앱 매니페스트를 통한 안드로이드 홈 화면 아이콘
- 투명 배경이 필요할 때

**한계:**
- 크기별로 별도의 파일이 필요함
- 무한 확대 불가 — 작은 PNG를 크게 확대하면 품질이 저하됨

---

### SVG — 확장 가능한 미래 형식

SVG(Scalable Vector Graphics)는 벡터 형식입니다. 픽셀 격자가 아닌 수학적 계산으로 정의되기 때문에, **어떤 크기나 해상도에서도 선명하게** 표시됩니다. 일반 디스플레이의 16×16이든, 고해상도 화면의 512×512이든 품질 저하가 없습니다.

최근 몇 년 사이 SVG 파비콘에 대한 브라우저 지원이 크게 늘었습니다. 크로미움 기반 브라우저(Chrome, Edge, Brave)와 Firefox는 SVG 파비콘을 지원하며, Safari와 일부 레거시 환경은 여전히 PNG와 ICO에 의존합니다.

SVG 파비콘의 특히 유용한 기능 중 하나는 **`prefers-color-scheme`** 지원입니다. CSS 미디어 쿼리를 활용해 사용자의 시스템 설정에 따라 라이트 모드와 다크 모드 아이콘을 자동으로 전환할 수 있습니다.

**SVG를 사용하는 경우:**
- 모든 화면 밀도에서 선명하게 표시되는 단일 파일이 필요할 때
- 라이트/다크 모드 자동 전환을 원할 때
- ICO, PNG와 함께 완전한 파비콘 셋업의 모던 구성 요소로

**한계:**
- 모든 브라우저에서 지원되지 않으므로 ICO/PNG 폴백 필수
- 사진처럼 래스터 기반 이미지는 SVG의 확장성 장점을 활용하기 어려움
- 올바르게 구현하려면 다소 복잡할 수 있음

## 세 가지 형식이 함께 작동하는 방식

어떤 형식도 단독으로 모든 브라우저와 기기를 완벽하게 커버하지 못합니다. 완성도 높은 파비콘 셋업은 세 가지 형식을 조합해 각자의 역할을 분담시키는 방식으로 구성됩니다.

| 파일 | 역할 |
|------|------|
| `favicon.ico` | 구형 브라우저를 위한 범용 폴백 |
| `favicon-16x16.png` | 모던 브라우저의 표준 탭 아이콘 |
| `favicon-32x32.png` | 고해상도 탭, 작업 표시줄 바로가기 |
| `apple-touch-icon.png` (180×180) | iOS 홈 화면 바로가기 |
| `android-chrome-192x192.png` | 안드로이드 홈 화면 |
| `android-chrome-512x512.png` | PWA 스플래시 화면 |
| `favicon.svg` | 모던 브라우저, 다크 모드 지원 |
| `site.webmanifest` | 안드로이드/PWA에 사용할 아이콘 정보 전달 |

## 필요한 HTML 코드

이 파일들을 웹사이트에 연결하려면 `<head>` 태그 안에 다음 코드를 추가하세요.

```html
<link rel="apple-touch-icon" sizes="180x180" href="/apple-touch-icon.png" />
<link rel="icon" type="image/png" sizes="32x32" href="/favicon-32x32.png" />
<link rel="icon" type="image/png" sizes="16x16" href="/favicon-16x16.png" />
<link rel="icon" type="image/svg+xml" href="/favicon.svg" />
<link rel="manifest" href="/site.webmanifest" />
<link rel="icon" href="/favicon.ico" sizes="any" />
```

순서도 중요합니다. 브라우저는 목록을 순서대로 확인하며 가장 적합한 형식을 선택합니다. `favicon.ico`를 `sizes="any"`와 함께 마지막에 배치하면, 위쪽의 더 구체적인 항목을 덮어쓰지 않으면서 폴백으로 작동합니다.

## 처음 파비콘을 만든다면?

파비콘을 처음 설정하면서 간단하게 시작하고 싶다면 다음 순서를 따르세요.

1. **고해상도 원본 이미지 준비** — 최소 512×512 픽셀 이상을 권장합니다.
2. **ICO와 PNG 크기 생성** — 대부분의 브라우저와 기기를 커버할 수 있습니다.
3. **가능하다면 SVG 추가** — 벡터 기반 아이콘이거나 다크 모드 지원을 원할 때 특히 유용합니다.
4. **webmanifest 포함** — 안드로이드 아이콘과 PWA 지원에 필수입니다.

**Favify** 같은 툴을 사용하면 이 과정이 자동으로 처리됩니다. 사진을 업로드하면 ICO, 전체 PNG 크기, webmanifest, 바로 붙여넣을 수 있는 HTML까지 완성된 패키지를 단 몇 초 만에 받을 수 있습니다.

## 정리

| 형식 | 확장 가능 | 투명도 | 브라우저 지원 | 최적 용도 |
|------|-----------|--------|---------------|-----------|
| ICO | ✗ | 부분 지원 | 전체 | 레거시 폴백 |
| PNG | ✗ | ✓ | 전체 | 모던 브라우저, Apple, Android |
| SVG | ✓ | ✓ | 모던 브라우저만 | 선명한 아이콘, 다크 모드 |

최고의 파비콘 셋업은 하나의 형식을 고르는 것이 아닙니다. 세 가지를 함께 사용해 어디서 표시되든 아이콘이 올바르게 보이도록 만드는 것입니다.
