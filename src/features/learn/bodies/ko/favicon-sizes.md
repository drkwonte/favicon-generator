파비콘 요구 사항을 찾아본 적이 있다면, 아마 끝도 없는 크기 목록에 당황했을 것입니다. 16×16, 32×32, 48×48, 57×57, 72×72, 96×96, 114×114, 120×120, 144×144, 152×152, 180×180, 192×192, 512×512... 목록은 계속 이어집니다. 머리가 아플 만도 합니다.

다행히 이 크기들을 전부 준비할 필요는 없습니다. 실제로 중요한 크기가 무엇인지, 각각 어디에 쓰이는지, 그리고 군더더기 없이 완전한 파비콘 셋업이 어떤 모습인지 명확하게 정리해드립니다.

---

## 왜 이렇게 크기가 많을까요?

파비콘은 생각보다 훨씬 다양한 곳에 등장합니다. 브라우저 탭, 북마크, 홈 화면 바로가기, PWA 스플래시 화면, 검색 결과까지 — 각 환경마다 요구하는 표시 크기가 다릅니다. 여기에 운영체제, 브라우저, 기기별로 저마다의 기준이 더해지면서 상황이 복잡해집니다.

결과적으로 단 하나의 크기로 모든 환경을 커버할 수 없는 상황이 만들어졌습니다. 해결책은 과도하게 많은 파일을 만드는 것이 아니라, 주요 사용 환경을 빠짐없이 커버하는 최적의 크기 조합을 선택하는 것입니다.

---

## 실제로 필요한 크기들

### 16×16 — 기본 탭 아이콘

가장 처음 등장한 파비콘 크기이자, 지금도 가장 중요한 크기입니다. 모든 데스크톱 브라우저는 브라우저 탭과 북마크 바에 16×16 아이콘을 표시합니다. 크기가 매우 작기 때문에, 이 크기에서도 읽힐 수 있도록 디자인은 최대한 단순해야 합니다.

**사용 환경:** 모든 데스크톱 브라우저 (Chrome, Firefox, Safari, Edge)
**권장 형식:** PNG 또는 ICO

---

### 32×32 — 고해상도 탭 및 작업 표시줄

맥의 레티나 디스플레이처럼 고해상도(고DPI) 화면을 사용하는 기기와 Windows 작업 표시줄 바로가기는 32×32 아이콘을 사용합니다. 이런 환경에서 브라우저는 더 선명한 렌더링을 위해 16×16 대신 32×32를 우선적으로 사용합니다.

**사용 환경:** 고DPI 브라우저, Windows 작업 표시줄 바로가기, macOS Safari
**권장 형식:** PNG 또는 ICO

---

### 48×48 — Windows 사이트 아이콘

Windows는 작업 표시줄이나 바탕화면에 고정된 사이트 바로가기에 48×48 아이콘을 사용합니다. 잘 언급되지 않는 크기지만, 완전한 Windows 호환성을 위해 `favicon.ico` 파일 안에 포함시켜 두는 것이 좋습니다.

**사용 환경:** Windows 바탕화면 및 작업 표시줄 바로가기
**권장 형식:** ICO (16×16, 32×32와 함께 내장)

---

### 180×180 — Apple Touch Icon

사용자가 아이폰이나 아이패드 홈 화면에 웹사이트를 저장할 때, iOS는 Apple Touch Icon을 앱 아이콘으로 사용합니다. 이 아이콘이 없으면 iOS는 페이지 스크린샷을 대신 사용하는데, 홈 화면 아이콘으로는 매우 보기 좋지 않습니다.

iOS 사용자가 많은 서비스라면 특히 신경 써야 할 크기입니다.

**사용 환경:** 아이폰, 아이패드 홈 화면 바로가기
**권장 형식:** PNG
**HTML 태그:** `<link rel="apple-touch-icon" sizes="180x180" href="/apple-touch-icon.png" />`

---

### 192×192 — 안드로이드 홈 화면

안드로이드 Chrome은 사용자가 사이트를 홈 화면에 추가할 때 192×192 아이콘을 사용합니다. 이 크기는 HTML에 직접 지정하는 것이 아니라 `site.webmanifest` 파일에 명시합니다.

**사용 환경:** 안드로이드 Chrome 홈 화면 바로가기
**권장 형식:** PNG
**지정 위치:** `site.webmanifest`

---

### 512×512 — PWA 스플래시 화면 및 고해상도 아이콘

512×512 아이콘은 프로그레시브 웹앱(PWA)이 실행될 때 표시되는 스플래시 화면에 사용됩니다. 일부 안드로이드 기기의 고해상도 홈 화면 아이콘에도 활용됩니다.

PWA를 구축 중이거나 계획 중이라면 필수 크기입니다. 그렇지 않더라도 포함해두면 향후 확장에 대비할 수 있습니다.

**사용 환경:** PWA 스플래시 화면, 고해상도 안드로이드 아이콘
**권장 형식:** PNG
**지정 위치:** `site.webmanifest`

---

## 최소한의 완전한 셋업

불필요한 파일 없이 완전한 커버리지를 원한다면, 실제로 필요한 파일은 다음과 같습니다.

| 파일 | 크기 | 용도 |
|------|------|------|
| `favicon.ico` | 16×16, 32×32, 48×48 | 범용 폴백, 모든 레거시 브라우저 |
| `favicon-16x16.png` | 16×16 | 모던 브라우저 탭 |
| `favicon-32x32.png` | 32×32 | 고DPI 탭, Windows 바로가기 |
| `apple-touch-icon.png` | 180×180 | iOS 홈 화면 |
| `android-chrome-192x192.png` | 192×192 | 안드로이드 홈 화면 |
| `android-chrome-512x512.png` | 512×512 | PWA 스플래시 화면 |
| `site.webmanifest` | — | 안드로이드/PWA 아이콘 설정 |

이 조합으로 현재 널리 사용되는 거의 모든 브라우저, 기기, 사용 환경을 커버할 수 있습니다.

## 나머지 크기들은 어떻게 할까요?

일부 파비콘 생성기는 57×57, 72×72, 114×114, 152×152 등 수십 개의 파일을 만들어냅니다. 이 크기들은 구형 iOS와 안드로이드를 위해 필요했지만, 오늘날에는 대부분 쓸모가 없어졌습니다.

건너뛰어도 되는 크기들을 정리하면 다음과 같습니다.

| 크기 | 원래 용도 | 지금도 필요한가? |
|------|-----------|-----------------|
| 57×57 | 아이폰 (레티나 이전, iOS 6 이하) | 불필요 |
| 72×72 | 아이패드 (초기 모델, iOS 6 이하) | 불필요 |
| 76×76 | 아이패드 (iOS 7) | 불필요 |
| 114×114 | 아이폰 레티나 (iOS 6 이하) | 불필요 |
| 120×120 | 아이폰 레티나 (iOS 7) | 불필요 |
| 144×144 | 아이패드 레티나 (iOS 6 이하) | 불필요 |
| 152×152 | 아이패드 레티나 (iOS 7) | 불필요 |
| 96×96 | 구형 안드로이드 / Google TV | 불필요 |

매우 오래된 iOS나 안드로이드 버전을 지원해야 할 특별한 이유가 없다면, 이 크기들은 모두 건너뛰어도 됩니다. 180×180 Apple Touch Icon이 모든 최신 iOS 기기를 커버하고, 192×192가 최신 안드로이드를 커버합니다.

## 원본 이미지 권장 사항

모든 크기는 결국 하나의 원본 이미지에서 출발합니다. 모든 크기에서 선명한 결과를 얻으려면 다음 사항을 염두에 두세요.

**최소 512×512 픽셀의 원본 이미지를 사용하세요.** 이 정도 해상도가 있어야 16×16 탭 아이콘까지 모든 크기로 깔끔하게 축소할 수 있습니다. 더 작은 이미지로 시작하면 작은 아이콘이 흐릿하거나 픽셀이 깨져 보일 수 있습니다.

**정사각형 이미지를 사용하세요.** 모든 파비콘 형식은 정사각형입니다. 원본 이미지가 정사각형이 아니라면, 대부분의 툴은 중앙을 기준으로 잘라내는데, 이 과정에서 중요한 부분이 잘릴 수 있습니다.

**디자인을 단순하게 유지하세요.** 512×512에서 멋지게 보이는 디테일이 16×16에서는 알아볼 수 없는 얼룩이 될 수 있습니다. 공개하기 전에 실제 탭 크기에서 아이콘을 반드시 확인하세요.

**가능하면 투명 배경을 사용하세요.** PNG 투명도를 활용하면 라이트 모드와 다크 모드 브라우저 테마 모두에서 아이콘이 자연스럽게 어우러집니다.

## HTML 코드

파일 준비가 끝났다면 `<head>` 태그 안에 다음 코드를 추가해 사이트에 연결하세요.

```html
<link rel="apple-touch-icon" sizes="180x180" href="/apple-touch-icon.png" />
<link rel="icon" type="image/png" sizes="32x32" href="/favicon-32x32.png" />
<link rel="icon" type="image/png" sizes="16x16" href="/favicon-16x16.png" />
<link rel="manifest" href="/site.webmanifest" />
<link rel="icon" href="/favicon.ico" sizes="any" />
```

그리고 `site.webmanifest`에는 다음 내용이 포함되어야 합니다.

```json
{
  "icons": [
    { "src": "/android-chrome-192x192.png", "sizes": "192x192", "type": "image/png" },
    { "src": "/android-chrome-512x512.png", "sizes": "512x512", "type": "image/png" }
  ]
}
```

## 정리

파비콘 크기가 30가지나 필요하지 않습니다. 필요한 건 올바른 크기들입니다. 완전한 현대적 파비콘 셋업은 결국 6개의 이미지 파일과 매니페스트 하나로 귀결됩니다. 불필요한 군더더기 없이 모든 주요 브라우저, 기기, 사용 환경을 커버할 수 있습니다.

**Favify**는 이 모든 것을 자동으로 생성합니다. 사진을 업로드하면 필요한 모든 크기의 파일이 올바른 이름으로 만들어지고, 바로 붙여넣을 수 있는 HTML 코드까지 단 몇 초 만에 받을 수 있습니다.
