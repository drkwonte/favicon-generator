아바타를 만들고 파비콘 패키지를 다운로드했습니다. 이제 활용할 차례입니다. 이 가이드는 가장 인기 있는 세 가지 웹사이트 플랫폼 — 워드프레스, 노션, 스쿼어스페이스 — 에 아바타 파비콘을 설치하는 방법을 단계별로 안내합니다.

다른 플랫폼을 사용하고 있다면 마지막 섹션에서 기타 일반적인 툴과 프레임워크에 대한 안내를 확인하세요.

---

## 시작하기 전에

파비콘 패키지가 준비되어 있는지 확인하세요. Favify 다운로드에는 필요한 모든 것이 포함되어 있습니다.

- `favicon.ico` — 모든 브라우저를 위한 범용 폴백
- `favicon-16x16.png` — 표준 브라우저 탭
- `favicon-32x32.png` — 고해상도 탭 및 Windows 바로가기
- `apple-touch-icon.png` — iOS 홈 화면 아이콘 (180×180)
- `android-chrome-192x192.png` — 안드로이드 홈 화면
- `android-chrome-512x512.png` — PWA 스플래시 화면
- `site.webmanifest` — 안드로이드 및 PWA 설정

패키지에는 바로 붙여넣을 수 있는 HTML 코드도 포함되어 있습니다. 일부 설치 방법에서 직접 사용하므로 가까이 두세요.

## 워드프레스

워드프레스는 세계에서 가장 널리 사용되는 웹사이트 플랫폼으로, 사이트 설정 방식에 따라 파비콘을 추가하는 여러 가지 방법을 제공합니다.

### 방법 1: 커스터마이저 사용하기 (대부분의 사용자에게 권장)

가장 간단한 방법으로 거의 모든 워드프레스 테마에서 작동합니다.

1. 워드프레스 관리자 대시보드에 로그인하세요.
2. **외모 → 사용자 정의**로 이동하세요.
3. 왼쪽 패널에서 **사이트 정체성**을 클릭하세요.
4. 스크롤을 내려 **사이트 아이콘** 섹션을 찾으세요.
5. **사이트 아이콘 선택**을 클릭하세요.
6. `android-chrome-512x512.png` 파일을 업로드하세요. 512×512 버전은 워드프레스에 가장 높은 품질의 원본을 제공합니다.
7. 워드프레스가 크롭 미리보기를 표시합니다. 필요하다면 조정한 후 **이미지 자르기**를 클릭하세요.
8. 변경 사항을 저장하려면 왼쪽 상단의 **공개**를 클릭하세요.

워드프레스가 자동으로 필요한 크기를 생성하고 적절한 HTML 태그를 사이트에 추가합니다. 코드를 수동으로 추가할 필요가 없습니다.

**참고:** 사이트 아이콘 기능은 활성 테마가 `add_theme_support( 'custom-logo' )`를 지원해야 합니다. 거의 모든 현대 테마가 지원합니다. 사이트 아이콘 옵션이 보이지 않으면 테마 문서를 확인하거나 방법 2를 사용하세요.

### 방법 2: 전체 코드 수동으로 추가하기 (완전한 제어를 위해)

ICO 파일, 모든 PNG 크기, webmanifest를 포함한 완전한 파비콘 패키지를 사용하려면 HTML 코드를 수동으로 추가해야 합니다. 이 방법은 완전한 제어를 제공하고 모든 브라우저와 기기가 커버되도록 보장합니다.

**1단계: 파비콘 파일 업로드하기**

FTP, 호스팅 제어판의 파일 관리자, 또는 WP File Manager 같은 플러그인을 사용해 모든 파비콘 파일을 워드프레스 설치의 루트 디렉토리 — `wp-config.php`가 있는 폴더 — 에 업로드하세요. 이것은 일반적으로 서버의 `/public_html/` 또는 `/www/` 폴더입니다.

**2단계: 테마에 HTML 코드 추가하기**

**외모 → 테마 파일 편집기**로 이동하세요 (테마 업데이트 시 변경 사항을 잃지 않으려면 자식 테마를 사용하세요). 테마의 `header.php` 파일을 열고 `<head>` 섹션을 찾으세요. 닫는 `</head>` 태그 바로 앞에 다음 코드를 추가하세요.

```html
<link rel="apple-touch-icon" sizes="180x180" href="/apple-touch-icon.png" />
<link rel="icon" type="image/png" sizes="32x32" href="/favicon-32x32.png" />
<link rel="icon" type="image/png" sizes="16x16" href="/favicon-16x16.png" />
<link rel="manifest" href="/site.webmanifest" />
<link rel="icon" href="/favicon.ico" sizes="any" />
```

**파일 업데이트**를 클릭해 저장하세요.

### 방법 3: 플러그인 사용하기

테마 파일 편집이 불편하다면 여러 플러그인이 파비콘 관리를 간단하게 해줍니다.

**Favicon by RealFaviconGenerator**는 가장 포괄적인 옵션 중 하나입니다. 완전한 파비콘 패키지를 생성하고 HTML을 자동으로 처리합니다.

**Insert Headers and Footers** (WPBeginner 제공)는 테마 파일을 건드리지 않고 사이트의 `<head>`에 HTML 코드를 붙여넣을 수 있게 해줍니다. 먼저 파비콘 파일을 서버 루트에 업로드한 다음 이 플러그인을 사용해 HTML 코드를 붙여넣으세요.

### 워드프레스 파비콘 확인하기

설치 후 브라우저에서 사이트를 방문하고 탭을 확인하세요. 파비콘이 즉시 나타나지 않으면 강력 새로고침(Windows: `Ctrl+Shift+R`, Mac: `Cmd+Shift+R`)을 하거나 시크릿 창에서 사이트를 여세요.

## 노션

노션의 파비콘 지원은 일반 웹사이트와 다르게 작동합니다. 시작하기 전에 이 차이점을 이해하는 것이 중요합니다.

### 노션이 실제로 지원하는 것

노션은 모든 페이지에 **페이지 아이콘**을 설정할 수 있습니다. 이 아이콘은 누군가 해당 페이지를 볼 때 브라우저 탭에 나타나며, 노션 사이드바와 링크 미리보기에도 나타납니다. 하지만 이것은 전통적인 파비콘이 아닙니다. 각 개별 페이지에 연결된 노션 특유의 기능입니다.

노션은 현재 커스텀 도메인에 사이트 전체에 적용되는 커스텀 `favicon.ico` 또는 PNG 파비콘 파일 업로드를 지원하지 **않습니다**. 노션의 내장 공유 기능을 사용해 노션 페이지를 웹에 게시한 경우 파비콘 동작은 설정에 따라 다릅니다.

- **기본 노션 공유** (notion.site 서브도메인): 파비콘은 노션이 제어하며 기본적으로 노션 로고를 표시합니다. 설정한 개별 페이지 아이콘은 해당 특정 페이지의 탭에 나타납니다.
- **서드파티 툴을 통한 커스텀 도메인** (Super, Potion, Fruition 등): 이러한 툴은 파비콘에 대한 완전한 제어권을 제공하며 표준 웹사이트처럼 완전한 Favify 패키지를 사용할 수 있습니다.

### 노션에서 페이지 아이콘 설정하기

노션 페이지의 아이콘으로 아바타를 설정하려면:

1. 노션에서 페이지를 여세요.
2. 페이지 제목 위로 마우스를 올려 **아이콘 추가** 버튼을 표시하거나, 이미 아이콘이 설정되어 있다면 기존 아이콘을 클릭하세요.
3. 아이콘 선택기에서 **업로드**를 클릭하세요.
4. `favicon-32x32.png` 또는 `apple-touch-icon.png`를 업로드하세요. 이 크기들은 노션 페이지 아이콘으로 잘 작동합니다.
5. 아이콘이 즉시 탭과 노션 사이드바에 나타납니다.

아바타를 아이콘으로 표시하려는 각 페이지에 대해 이 과정을 반복하세요.

### 커스텀 노션 도메인으로 전체 파비콘 사용하기

**Super** (super.so), **Potion** (potion.so), 또는 유사한 툴을 사용해 커스텀 도메인에 노션 사이트를 게시하는 경우 파비콘에 대한 완전한 제어권을 가집니다.

**Super 사용 시:**
1. Super 대시보드로 이동해 사이트를 선택하세요.
2. **설정 → Favicon**으로 이동하세요.
3. 파비콘 이미지를 업로드하세요 (최상의 결과를 위해 512×512 PNG 사용).
4. 저장하고 사이트를 다시 게시하세요.

Super가 HTML을 자동으로 처리합니다. 완전한 제어를 원한다면 Super의 **코드 삽입** 기능을 통해 커스텀 코드를 추가하고 전체 HTML 코드를 붙여넣을 수도 있습니다.

**Potion 사용 시:**
1. Potion 대시보드로 이동하세요.
2. 사이트를 선택하고 **설정**으로 이동하세요.
3. **사이트 아이콘** 섹션에서 파비콘을 업로드하세요.
4. 변경 사항을 저장하세요.

**Fruition 또는 커스텀 Cloudflare Workers 설정 사용 시:**
Cloudflare Worker 스크립트에서 페이지 `<head>` 콘텐츠가 삽입되는 섹션에 HTML 코드를 추가하세요. 파비콘 파일을 CDN이나 정적 호스트에 업로드하고 코드에서 전체 URL로 참조하세요.

## 스쿼어스페이스

스쿼어스페이스에는 사용하기 간단한 내장 파비콘 기능이 있지만, 완전한 커스텀 HTML 설정과 비교했을 때 몇 가지 제한이 있습니다.

### 스쿼어스페이스 설정을 통해 파비콘 추가하기

1. 스쿼어스페이스 계정에 로그인하고 사이트를 여세요.
2. **웹사이트 → 웹사이트 도구 → 파비콘**으로 이동하세요.

   *구버전 스쿼어스페이스(7.0):* **디자인 → 브라우저 아이콘(파비콘)**으로 이동하세요.

3. **업로드**를 클릭하고 파비콘 이미지를 선택하세요.
   - 최상의 품질을 위해 `apple-touch-icon.png` (180×180)를 사용하세요. 스쿼어스페이스는 정사각형 이미지를 허용하고 내부적으로 크기 조정을 처리합니다.
   - 또는 가장 높은 해상도 원본을 위해 `android-chrome-512x512.png`를 사용하세요.
4. 업로드되면 브라우저 탭에서 파비콘이 어떻게 나타날지 미리보기가 표시됩니다.
5. **저장**을 클릭해 변경 사항을 적용하세요.
6. 파비콘은 몇 분 내에 사이트에 적용됩니다.

### 스쿼어스페이스 파비콘의 제한사항

스쿼어스페이스의 내장 파비콘 시스템에는 알아두어야 할 몇 가지 제약이 있습니다.

**단일 이미지 업로드만 가능합니다.** 스쿼어스페이스는 업로드한 하나의 이미지에서 다양한 파비콘 크기를 생성합니다. ICO, PNG, webmanifest 파일을 개별적으로 업로드할 수 없습니다. 대부분의 사용자에게는 괜찮습니다. 스쿼어스페이스가 일반적인 경우를 잘 처리합니다.

**webmanifest 제어가 없습니다.** 스쿼어스페이스는 `site.webmanifest` 파일에 대한 직접적인 제어를 제공하지 않습니다. 즉, 안드로이드 PWA 및 홈 화면 아이콘 설정이 당신이 아닌 플랫폼에 의해 자동으로 처리됩니다.

**직접 ICO 업로드가 없습니다.** 스쿼어스페이스는 내부적으로 PNG를 사용합니다. 레거시 브라우저를 위한 ICO 폴백은 플랫폼이 처리합니다.

대부분의 스쿼어스페이스 사이트에는 내장 파비콘 기능으로 충분합니다. PWA 지원이나 커스텀 안드로이드 홈 화면 아이콘 같이 더 많은 제어가 필요하다면 스쿼어스페이스의 **코드 삽입** 기능을 사용해야 합니다.

### 고급: 전체 파비콘 제어를 위한 코드 삽입 사용하기

스쿼어스페이스 Business 및 Commerce 플랜은 커스텀 코드 삽입을 허용하여 파비콘 설정에 대한 완전한 제어권을 줍니다.

1. **설정 → 고급 → 코드 삽입**으로 이동하세요.
2. **헤더** 섹션에 전체 HTML 파비콘 코드를 붙여넣으세요.

```html
<link rel="apple-touch-icon" sizes="180x180" href="YOUR_FILE_URL/apple-touch-icon.png" />
<link rel="icon" type="image/png" sizes="32x32" href="YOUR_FILE_URL/favicon-32x32.png" />
<link rel="icon" type="image/png" sizes="16x16" href="YOUR_FILE_URL/favicon-16x16.png" />
<link rel="manifest" href="YOUR_FILE_URL/site.webmanifest" />
<link rel="icon" href="YOUR_FILE_URL/favicon.ico" sizes="any" />
```

3. `YOUR_FILE_URL`을 파비콘 파일을 호스팅한 URL로 교체하세요. 스쿼어스페이스는 직접적인 파일 시스템 접근을 제공하지 않으므로 파비콘 파일을 외부 — CDN, GitHub Pages, 또는 다른 정적 호스팅 서비스 — 에 호스팅하고 전체 URL로 참조해야 합니다.
4. **저장**을 클릭하세요.

### 스쿼어스페이스 파비콘 확인하기

저장 후 사이트를 방문하고 브라우저 탭을 확인하세요. 스쿼어스페이스가 변경 사항을 적용하는 데 몇 분이 걸릴 수 있습니다. 이전 파비콘이 계속 나타나면 강력 새로고침을 하거나 시크릿 창에서 확인하세요.

## 기타 플랫폼

### Webflow

1. **프로젝트 설정 → 일반**으로 이동하세요.
2. **Favicon & App Icon**으로 스크롤하세요.
3. 정사각형 PNG를 업로드하세요 (512×512 권장).
4. **변경 사항 저장**을 클릭하고 사이트를 게시하세요.

완전한 제어를 위해 **프로젝트 설정 → 커스텀 코드 → 헤드 코드**를 사용해 HTML 코드를 붙여넣으세요. Webflow의 에셋 관리자나 외부 CDN을 통해 파비콘 파일을 호스팅하세요.

### Ghost

1. Ghost 관리자 패널로 이동하세요.
2. **설정 → 디자인 → 브랜드**로 이동하세요.
3. **사이트 아이콘** 항목에 파비콘 이미지를 업로드하세요.
4. 변경 사항을 저장하세요.

완전한 파비콘 패키지를 위해 **설정 → 코드 삽입** 아래의 Ghost 코드 삽입 기능을 사용하고 사이트 헤더 필드에 HTML 코드를 붙여넣으세요. 파일을 외부 호스트에 업로드하고 URL로 참조하세요.

### Wix

1. Wix 대시보드로 이동하세요.
2. **설정 → 파비콘**을 클릭하세요.
3. **파비콘 업로드**를 클릭하고 이미지를 선택하세요.
4. **저장**을 클릭하세요.

Wix는 커스텀 파비콘 업로드를 직접 지원합니다. 고급 제어를 위해 **설정 → 커스텀 코드** 아래의 Wix 커스텀 코드 기능(유료 플랜에서 사용 가능)을 사용하세요.

### Shopify

1. Shopify 관리자로 이동하세요.
2. **온라인 스토어 → 테마**로 이동하세요.
3. 활성 테마에서 **사용자 정의**를 클릭하세요.
4. **테마 설정 → 파비콘**으로 이동하세요.
5. 파비콘 이미지를 업로드하고 저장하세요.

전체 패키지를 위해 테마의 `theme.liquid` 파일을 편집하고 `<head>` 태그 안에 HTML 코드를 추가하세요. **콘텐츠 → 파일**을 통해 Shopify의 파일 호스팅에 파비콘 파일을 업로드하세요.

### GitHub Pages 및 정적 사이트

모든 파비콘 파일을 저장소 루트(또는 Next.js, Astro, Hugo 같은 프레임워크의 `/public` 폴더)에 배치하세요. 베이스 레이아웃 템플릿이나 `index.html` 파일의 `<head>` 태그 안에 HTML 코드를 추가하세요.

```html
<link rel="apple-touch-icon" sizes="180x180" href="/apple-touch-icon.png" />
<link rel="icon" type="image/png" sizes="32x32" href="/favicon-32x32.png" />
<link rel="icon" type="image/png" sizes="16x16" href="/favicon-16x16.png" />
<link rel="manifest" href="/site.webmanifest" />
<link rel="icon" href="/favicon.ico" sizes="any" />
```

변경 사항을 커밋하고 푸시하세요. GitHub Pages가 사이트를 재빌드하면 파비콘이 적용됩니다.

## 일반적인 문제 해결

**파비콘이 나타나지 않습니다.**
파일 경로가 올바른지 확인하세요. HTML의 경로와 실제 파일 위치 간의 불일치가 가장 흔한 원인입니다. 강력 새로고침(`Ctrl+Shift+R` / `Cmd+Shift+R`)을 시도하거나 시크릿 창에서 사이트를 여세요.

**이전 파비콘이 계속 나타납니다.**
브라우저는 파비콘을 적극적으로 캐싱합니다. 파일을 업데이트한 후에도 브라우저가 몇 시간 동안 이전 것을 계속 표시할 수 있습니다. 시크릿 창은 이 캐시를 우회하며 현재 파비콘을 즉시 표시합니다.

**파비콘이 흐릿하거나 픽셀이 깨져 보입니다.**
일반적으로 저해상도 원본 이미지를 업로드했다는 의미입니다. Favify 패키지의 `android-chrome-512x512.png` 또는 다른 고해상도 파일을 사용해 다시 업로드하세요.

**Chrome에서는 파비콘이 나타나지만 Safari에서는 나타나지 않습니다.**
Safari는 자체 파비콘 캐싱 동작을 가지고 있으며 때때로 업데이트하는 데 더 오래 걸립니다. **개발자 → 캐시 비우기**를 통해 Safari의 캐시를 지워보세요 (보이지 않으면 Safari 환경설정에서 개발자 메뉴를 활성화하세요).

**탭에는 파비콘이 표시되지만 iOS 홈 화면에는 나타나지 않습니다.**
`apple-touch-icon.png` (180×180)이 올바르게 업로드되어 있고 HTML에 `<link rel="apple-touch-icon">` 태그가 있는지 확인하세요. 이 태그가 없으면 iOS는 페이지 스크린샷을 대신 사용합니다.
